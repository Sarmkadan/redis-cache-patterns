# RequestLoggingMiddleware

Middleware component that logs incoming HTTP requests for diagnostic and auditing purposes. It captures essential request details such as the HTTP method, operation name, and a unique request identifier to correlate logs with specific requests across distributed systems.

## API

### `RequestLoggingMiddleware`

Constructor for the middleware component. Initializes the middleware with default or injected dependencies for request logging.

**Parameters:**
- None

**Remarks:**
- Typically registered in the ASP.NET Core pipeline via `app.UseMiddleware<RequestLoggingMiddleware>()`.

---

### `InvokeAsync`

Invokes the middleware to process an HTTP request and log its details.

**Parameters:**
- `HttpContext context`
  The HTTP context containing the request to be logged.

**Return value:**
- `Task`
  A task that represents the asynchronous operation.

**Remarks:**
- Logs the request method, operation name, and request ID at the start of request processing.
- May throw if logging infrastructure fails (e.g., disk full, network issues in remote logging).

---

### `RequestId`

Gets the unique identifier for the current request, used to correlate logs.

**Type:** `string?`

**Remarks:**
- Corresponds to the `RequestId` header or value generated by ASP.NET Core.
- May be `null` if the request ID is not available.

---

### `Method`

Gets the HTTP method (e.g., GET, POST) of the current request.

**Type:** `string`

**Remarks:**
- Always non-null; derived from `HttpRequest.Method`.

---
### `OperationName`

Gets the name of the operation being performed, typically derived from routing or metadata.

**Type:** `string`

**Remarks:**
- May be `null` if the operation name cannot be determined (e.g., unmatched route).
