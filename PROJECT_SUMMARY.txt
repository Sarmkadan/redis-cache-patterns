FEATURES IMPLEMENTED
================================================================================

1. CACHE-ASIDE PATTERN
   - Check cache first
   - Load from database on miss
   - Automatically store in cache
   - Configurable TTL per operation
   - Used by: User retrieval, Product lookups, Order queries

2. WRITE-THROUGH PATTERN
   - Persist to database first
   - Update cache immediately after
   - Atomic operation (both succeed or both fail)
   - Ensures cache consistency
   - Used by: User creation/update, Product updates, Order operations

3. DISTRIBUTED LOCKS
   - Prevent race conditions across instances
   - Automatic lock expiration
   - Lock renewal mechanism
   - Lock value validation (only holder can release)
   - Used by: Order confirmation, Inventory operations

4. CACHE INVALIDATION
   - Single key deletion
   - Pattern-based key deletion (prefix matching)
   - Automatic invalidation on related operations
   - Tag-based tracking for complex invalidation

5. ERROR HANDLING
   - Custom exception hierarchy
   - Specific error codes for different failures
   - Proper logging at each layer
   - Graceful fallback on Redis unavailability

6. MONITORING & DIAGNOSTICS
   - Cache statistics (keys, memory, hit rate)
   - Per-entry hit tracking
   - Cold entry detection
   - Performance timing

CACHING PATTERNS DEMONSTRATED
================================================================================

1. Cache-Aside Pattern
   ├─ Scenario: User retrieval
   ├─ Flow: Check Cache → Miss → Load DB → Store Cache → Return
   └─ Service: UserService.GetUserByIdAsync()

2. Write-Through Pattern
   ├─ Scenario: User creation
   ├─ Flow: Persist DB → Update Cache → Return
   └─ Service: UserService.CreateUserAsync()

3. Distributed Lock Pattern
   ├─ Scenario: Order confirmation
   ├─ Flow: Acquire Lock → Process → Release Lock
   └─ Service: OrderService.ConfirmOrderAsync()

4. Cache Invalidation Pattern
   ├─ Scenario: Product update
   ├─ Flow: Update → Invalidate Related Keys
   └─ Service: ProductService.UpdateProductAsync()
